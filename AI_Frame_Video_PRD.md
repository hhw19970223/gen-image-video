# AI 帧动画视频生成平台 PRD

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | AI 帧动画视频生成平台 |
| 文档版本 | v1.0 |
| 文档类型 | PRD / 产品需求文档 |
| 目标阶段 | MVP 到可商业验证版本 |
| 核心方向 | 通过“关键帧生图 + 插帧/动画 + 视频合成 + 缓存复用”低成本生成短视频 |

## 2. 产品背景

当前主流 AI 生视频模型对显卡要求较高，普通个人电脑很难本地稳定运行。以当前设备配置为例，CPU 为 Intel i5-10400、内存 32GB、显卡为 Intel UHD Graphics 630 核显，缺少 NVIDIA CUDA 独立显卡，不适合直接运行 Wan、LTX、CogVideoX、HunyuanVideo 等重型生视频模型。

因此，本项目采用更轻量、更可控的技术路线：先生成关键帧图片，再通过动画、插帧和 FFmpeg 合成为视频。系统通过缓存关键帧、中间帧和最终视频，减少重复生成成本，并为后续接入云 GPU、ComfyUI、第三方生图/生视频 API 留出扩展接口。

## 3. 产品目标

### 3.1 核心目标

构建一个可以根据用户输入的提示词、图片、分镜和视频参数，自动生成短视频的 Web 平台。

第一版目标不是生成电影级真实运动视频，而是跑通一个稳定、低成本、可复用的 AI 视频生成闭环。

### 3.2 业务目标

- 降低 AI 视频生成对本地硬件的依赖。
- 通过缓存机制降低重复生成成本。
- 支持面向商品展示、短视频素材、社媒广告、图文转视频等场景。
- 为后续接入更强的视频生成模型预留架构。

### 3.3 用户目标

- 用户输入一句描述或上传一张图片，即可生成一段短视频。
- 用户可以查看和修改关键帧。
- 用户可以重新生成单帧，而不是重新生成整个视频。
- 用户可以下载最终 mp4 视频。
- 用户可以复用历史素材，减少等待时间和生成费用。

## 4. 目标用户

### 4.1 初期用户

- 电商运营人员
- 短视频创作者
- 广告素材制作人员
- 小型品牌主
- AI 工具爱好者

### 4.2 典型使用场景

| 场景 | 描述 |
| --- | --- |
| 商品展示视频 | 输入商品描述或上传商品图，生成商品旋转、推进、开箱类短视频 |
| 社媒短视频素材 | 生成 5 到 15 秒的 TikTok、抖音、小红书素材 |
| 图文转视频 | 根据一段文案生成多张关键帧并合成为视频 |
| 广告素材批量生成 | 同一商品生成多个风格、比例和文案版本 |
| 品牌模板视频 | 固定品牌颜色、Logo、字体和片尾样式 |

## 5. 产品范围

### 5.1 MVP 范围

MVP 需要完成以下闭环：

1. 用户创建视频生成任务。
2. 用户输入提示词或上传参考图。
3. 系统生成 4 到 8 张关键帧。
4. 系统对关键帧做简单动画或插帧。
5. 系统使用 FFmpeg 合成 mp4 视频。
6. 系统保存任务、参数、关键帧、中间帧和最终视频。
7. 用户可以预览和下载视频。
8. 系统可以命中缓存，避免重复生成。

### 5.2 非 MVP 范围

以下功能不进入第一版：

- 多用户计费系统
- 复杂团队协作
- 高级视频模型本地部署
- 实时视频生成
- 复杂音频生成
- 移动端 App
- 完整素材市场

## 6. 产品流程

### 6.1 主流程

```text
用户进入创建页面
  -> 输入提示词 / 上传参考图
  -> 选择比例、时长、风格、帧率
  -> 点击生成
  -> 后端创建任务
  -> 检查缓存
  -> 生成或复用关键帧
  -> 生成中间帧 / 动画帧
  -> FFmpeg 合成视频
  -> 返回预览和下载链接
```

### 6.2 单帧重生成流程

```text
用户打开任务详情
  -> 选择不满意的关键帧
  -> 修改该帧提示词或随机种子
  -> 重新生成单帧
  -> 系统只重算受影响的中间帧
  -> 重新合成最终视频
```

### 6.3 缓存复用流程

```text
用户提交任务
  -> 系统计算参数 hash
  -> 查询关键帧缓存
  -> 查询中间帧缓存
  -> 查询最终视频缓存
  -> 命中则直接复用
  -> 未命中则生成并写入缓存
```

## 7. 功能需求

### 7.1 创建视频任务

用户可以创建一个视频任务。

字段包括：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| prompt | 文本 | 是 | 视频主题描述 |
| reference_image | 图片 | 否 | 参考图 |
| style | 枚举 | 否 | 写实、卡通、产品摄影、赛博朋克等 |
| aspect_ratio | 枚举 | 是 | 9:16、16:9、1:1 |
| duration | 数字 | 是 | 视频时长，MVP 支持 5 到 10 秒 |
| fps | 数字 | 是 | MVP 默认 24fps |
| frame_count | 数字 | 是 | 关键帧数量，默认 4 到 8 张 |
| seed | 数字 | 否 | 随机种子 |
| motion_type | 枚举 | 否 | 推进、拉远、平移、旋转、淡入淡出 |

验收标准：

- 用户可以提交任务。
- 参数非法时给出明确提示。
- 任务创建后进入排队或生成状态。

### 7.2 关键帧生成

系统根据用户输入生成关键帧。

需求：

- 支持调用 ComfyUI API、第三方生图 API 或本地图片生成服务。
- 每张关键帧保存原图、缩略图、生成参数和 seed。
- 支持单独重生成某一帧。
- 支持锁定某一帧，锁定后不再自动覆盖。

验收标准：

- 一个任务可以生成至少 4 张关键帧。
- 关键帧生成失败时不影响已完成帧。
- 用户可以查看每张关键帧。

### 7.3 简单动画生成

MVP 先支持低成本动画，不强依赖重型视频模型。

动画类型：

| 类型 | 说明 |
| --- | --- |
| zoom_in | 镜头缓慢推进 |
| zoom_out | 镜头缓慢拉远 |
| pan_left | 向左平移 |
| pan_right | 向右平移 |
| fade | 淡入淡出 |
| crossfade | 两张关键帧交叉溶解 |

验收标准：

- 不依赖独立显卡也能生成基础动画视频。
- 生成的视频画面连续，没有明显黑帧。
- 支持 9:16、16:9、1:1 三种比例。

### 7.4 插帧能力

MVP 可以先预留接口，第二阶段接入 RIFE 或云端插帧服务。

需求：

- 插帧服务作为可替换模块。
- 支持开启或关闭插帧。
- 插帧失败时可以降级为普通转场。

验收标准：

- 开启插帧时，系统可以调用插帧模块。
- 插帧异常时任务不会整体崩溃。

### 7.5 视频合成

使用 FFmpeg 将图片序列合成为 mp4。

需求：

- 支持输出 mp4。
- 支持设置 fps、分辨率、码率。
- 支持视频封面图。
- 支持任务完成后生成预览地址。

验收标准：

- 生成的视频可以在浏览器中播放。
- 视频时长与用户选择基本一致。
- 视频文件可以下载。

### 7.6 缓存系统

缓存是本项目的核心能力。

缓存对象：

| 缓存对象 | 说明 |
| --- | --- |
| prompt_cache | 文案和分镜结果 |
| keyframe_cache | 关键帧图片 |
| motion_frame_cache | 动画中间帧 |
| video_cache | 最终视频 |

缓存 key 示例：

```text
hash(
  prompt +
  reference_image_hash +
  model +
  style +
  seed +
  width +
  height +
  frame_index +
  motion_type +
  duration +
  fps
)
```

验收标准：

- 同参数重复生成时可以命中缓存。
- 单帧变更时只重算受影响部分。
- 任务详情中可以显示缓存命中状态。

### 7.7 任务状态管理

任务状态：

| 状态 | 说明 |
| --- | --- |
| pending | 等待中 |
| generating_keyframes | 生成关键帧中 |
| generating_motion | 生成动画帧中 |
| composing_video | 合成视频中 |
| completed | 已完成 |
| failed | 失败 |
| cancelled | 已取消 |

验收标准：

- 前端可以实时或轮询显示任务状态。
- 失败任务可以查看错误原因。
- 用户可以重新运行失败任务。

### 7.8 任务历史

需求：

- 展示历史任务列表。
- 支持按状态筛选。
- 支持查看任务详情。
- 支持复制历史任务参数重新生成。

验收标准：

- 用户可以找到之前生成的视频。
- 历史任务不会因为页面刷新丢失。

## 12. 生成策略

### 12.1 MVP 生成策略

MVP 阶段优先保证稳定性：

1. 先生成关键帧。
2. 对关键帧应用基础动画。
3. 使用转场连接不同关键帧。
4. 用 FFmpeg 输出 mp4。

### 12.2 后续增强策略

后续可以逐步增强：

- 接入 RIFE 插帧。
- 接入 ComfyUI 视频工作流。
- 接入 Wan / LTX / CogVideoX 云端推理。
- 加入 ControlNet / IP-Adapter 保持角色或商品一致性。
- 加入背景音乐和字幕。

## 13. 性能要求

MVP 目标：

| 指标 | 要求 |
| --- | --- |
| 单任务视频时长 | 5 到 10 秒 |
| 输出分辨率 | MVP 支持 720p 或以下 |
| 任务状态刷新 | 2 到 5 秒内更新 |
| 视频合成时间 | 普通短视频 1 分钟内完成，不含生图时间 |
| 缓存命中响应 | 10 秒内返回可预览结果 |

## 14. 风险与应对

| 风险 | 说明 | 应对 |
| --- | --- | --- |
| 本地显卡不足 | 当前电脑无法跑重型生视频 | 使用云端推理或远程 ComfyUI |
| 关键帧不一致 | 多张图之间角色或商品变化 | 使用参考图、固定 seed、IP-Adapter |
| 动作不自然 | 简单动画不如真视频 | MVP 接受，后续接插帧和视频模型 |
| 生成成本高 | 多次重复生成费用高 | 做缓存、单帧重算、参数复用 |
| 任务失败率高 | 外部生成服务不稳定 | 任务重试、失败降级、错误提示 |

## 16. MVP 验收标准

MVP 完成时应满足：

- 用户可以创建一个视频生成任务。
- 系统可以生成或复用至少 4 张关键帧。
- 系统可以将关键帧合成为一个 mp4 视频。
- 用户可以预览和下载视频。
- 用户可以查看任务状态。
- 用户可以重新生成某一张关键帧。
- 系统可以缓存重复生成结果。
- 即使没有独立显卡，本地仍可完成任务管理、缓存和视频合成流程。

## 18. 一句话总结

本项目不是直接硬跑大模型生视频，而是通过“关键帧生成、逐帧动画、插帧增强、视频合成、缓存复用”的组合方式，先做出一个普通电脑也能开发和运营的低成本 AI 视频生成系统。
